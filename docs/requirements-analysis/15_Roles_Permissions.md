## آلية التنفيذ

### على مستوى الـ Middleware
- فحص الجلسة (NextAuth token)
- توجيه المستخدمين غير المصادقين إلى `/login`
- توجيه كل دور إلى صفحته الافتراضية

### على مستوى الـ API
- `requireAuth(request, ['admin', 'manager'])` — يُرجع 401/403
- فلترة البيانات حسب الدور (SQL-level)
- إزالة الأعمدة الحساسة (buy_price للبائعين)
