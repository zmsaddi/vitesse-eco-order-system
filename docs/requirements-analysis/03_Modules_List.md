# قائمة الوحدات الوظيفية — Modules List

> **رقم العنصر**: #1
> **المحور**: أ — الأساسيات والهيكل العام
> **الحالة**: مكتمل

---

## ملخص

النظام يتكون من **15 وحدة وظيفية** موزعة على **5 مجموعات تنقل** في القائمة الجانبية.

---

## هيكل التنقل (Sidebar Groups)

```
├── لوحة التحكم (بدون مجموعة)
├── عمليات  ← المشتريات، المبيعات، المصاريف، التوصيل
├── مالية   ← الفواتير، التسويات، توزيع الأرباح
├── بيانات  ← المخزون، العملاء، الموردين، العمولة
└── نظام    ← الإعدادات، المستخدمين
```

## قائمة الوحدات التفصيلية

### المجموعة 1: لوحة التحكم

#### 1. لوحة التحكم — Summary Dashboard

| البند | التفصيل |
|-------|---------|
| المسار | `/summary` |
| الأدوار | admin, manager, seller |
| الغرض | عرض شامل لمؤشرات الأداء المالي والتجاري |

**الميزات الرئيسية:**
- بطاقات KPI: الإيرادات، صافي الربح، الديون، الربح النقدي
- تحليل الأرباح والخسائر (3 طرق): Pipeline / Accrual / Cash
- جدول التدفق النقدي: حركة الأموال الفعلية مقسمة كاش/بنك
- تبويب التقارير: رسوم بيانية (مبيعات 6 أشهر، مصاريف دائرية، اتجاه الأرباح)
- تصنيف المخزون حسب الفئة (admin/manager فقط)
- قوائم أعلى: المدينون، المنتجات، البائعون، الموردون
- أزرار سريعة: +عملية بيع، +عملية شراء
- تصدير CSV
- فلاتر التاريخ: الشهر الحالي، الشهر السابق، السنة، الكل، مخصص
- دعم الإدخال الصوتي (admin/manager/seller)

**عرض حسب الدور:**
- البائع: إحصائيات شخصية فقط
- admin/manager: العرض الكامل

### المجموعة 2: عمليات

#### 2. المبيعات — Sales

| البند | التفصيل |
|-------|---------|
| المسار | `/sales` |
| الأدوار | admin, manager, seller, driver (السائق يرى المرتبطة بتسليماته) |
| الغرض | تسجيل وإدارة عمليات البيع للعملاء |

**الميزات الرئيسية:**
- إضافة بيع جديد: اختيار عميل + منتج + كمية + سعر + طريقة دفع
- إنشاء عميل تلقائي إذا لم يكن موجودًا
- التحقق من السعر: البائع لا يبيع تحت sell_price، لا أحد يبيع تحت buy_price
- إدارة الحالات: محجوز → مؤكد → ملغي
- تتبع حالة الدفع: pending → partial → paid
- تعيين التوصيل تلقائيًا عند الإنشاء
- سير عمل الإلغاء (FEAT-05): إلغاء كامل مع معالجة العمولات
- فلاتر: نطاق تاريخ، اسم العميل، الحالة، حالة الدفع، البائع (admin فقط)
- إجراءات: عرض، تعديل، توليد فاتورة PDF، إلغاء، حذف

**قيود الدور:**
- البائع: لا يرى أعمدة التكلفة/الربح
- admin: يعدل حتى المبيعات المؤكدة
- manager: يعدل المحجوزة فقط

#### 3. المشتريات — Purchases

| البند | التفصيل |
|-------|---------|
| المسار | `/purchases` |
| الأدوار | admin, manager |
| الغرض | تسجيل المشتريات من الموردين وإعادة تعبئة المخزون |

**الميزات الرئيسية:**
- إضافة مشترى جديد: مورد + منتج + فئة + كمية + سعر + دفعة أولى
- تعديل/حذف المشتريات
- إدارة الدفعات الجزئية للموردين: زر "دفع" → PaySupplierDialog
- حالة الدفع: paid / partial / pending
- نظام ائتمان الموردين: الدفعة الأولى تنشئ دين المورد
- التحقق من سعر البيع: تحذير إذا sell_price < buy_price
- فلاتر: نطاق تاريخ، مورد، بحث، حالة الدفع
- بطاقات ملخص: إجمالي المشتريات، المدفوع، دين الموردين

#### 4. التوصيل — Deliveries

| البند | التفصيل |
|-------|---------|
| المسار | `/deliveries` |
| الأدوار | admin, manager, seller (قراءة), driver (تحديث الخاصة) |
| الغرض | إدارة تنفيذ الطلبات وتعيين السائقين وتأكيد التسليم |

**الميزات الرئيسية:**
- إضافة توصيل جديد: تاريخ + عميل + عنوان + أصناف + تعيين سائق
- إدارة الحالات: قيد الانتظار → جاري التوصيل → تم التوصيل / ملغي
- نافذة تأكيد VIN: تظهر تلقائيًا للدراجات الكهربائية (اكتشاف تلقائي)
- تعيين السائق (admin/manager فقط)
- تأكيد الحالة بنافذة تأكيد
- سير عمل الإلغاء: يفتح CancelSaleDialog
- فلاتر: نطاق تاريخ، الحالة، تعيين السائق
- السائق: يرى ويحدّث تسليماته المعينة فقط

### المجموعة 3: مالية

#### 6. الفواتير — Invoices

| البند | التفصيل |
|-------|---------|
| المسار | `/invoices` |
| الأدوار | admin, manager, seller, driver (المرتبطة فقط) |
| الغرض | عرض الفواتير المُولّدة تلقائيًا عند تأكيد التسليم — سجل تدقيق غير قابل للتعديل |

**الميزات الرئيسية:**

- استخراج حالة الدفع: مدفوع / جزئي / معلق / ملغي
- بحث: بالعميل، الكود المرجعي، المنتج، VIN
- تحميل PDF: فتح الفاتورة في تبويب جديد للطباعة
- عرض تفاصيل: نافذة تفصيلية كاملة

#### 7. التسويات — Settlements

| البند | التفصيل |
|-------|---------|
| المسار | `/settlements` |
| الأدوار | admin فقط |
| الغرض | تسجيل دفعات العمولات والرواتب للبائعين والسائقين |

**الميزات الرئيسية:**
- إضافة تسوية: نوع (بائع/سائق) + مستخدم + وصف + مبلغ + ملاحظات
- التحقق المباشر من الرصيد: يجلب المستخدمين المؤهلين مع الرصيد المتاح
- يمنع الدفع الزائد: المبلغ ≤ الرصيد المتاح
- نظام المستخدمين المؤهلين: قائمة ديناميكية حسب نوع التسوية
- جدول التسويات: قابل للترتيب والتقسيم
- شريط جانبي: العمولات غير المصروفة لكل مستخدم

#### 8. توزيع الأرباح — Profit Distributions

| البند | التفصيل |
|-------|---------|
| المسار | `/profit-distributions` |
| الأدوار | admin (إنشاء)، admin + manager (عرض) |
| الغرض | توزيع صافي الأرباح النقدية على المساهمين بالنسب المئوية |

**الميزات الرئيسية:**
- إنشاء توزيع: مبلغ أساسي + فترة + مستلمين متعددين مع نسب
- حساب حوض الفترة: صافي الربح النقدي، الموزع سابقًا، المتبقي
- إدارة حصص المشاركة: نسب مُعدّة مسبقًا لكل مستخدم
- زر "تحميل من الإعدادات": تعبئة تلقائية من الإعداد المسبق
- التحقق: النسب يجب أن تكون = 100%
- جدول التوزيعات السابقة
- المستخدمون المؤهلون: admin و manager فقط

### المجموعة 4: بيانات

#### 9. المخزون — Stock / Inventory

| البند | التفصيل |
|-------|---------|
| المسار | `/stock` |
| الأدوار | admin, manager, seller (عرض محدود) |
| الغرض | تتبع مستويات المخزون وإدارة المنتجات والأسعار |

**الميزات الرئيسية:**
- إدارة المنتجات: اسم + فئة + وصف عربي + سعر شراء + سعر بيع
- حالة المخزون لكل منتج: متوفر / منخفض / نفذ (حسب حد التنبيه)
- تعديل سعر البيع المباشر مع نافذة تأكيد
- فلاتر: بحث، حالة المخزون، الفئة
- حذف المنتج (admin فقط)
- بطاقات ملخص: عدد المنتجات، إجمالي المخزون، قيمة المخزون

**عرض حسب الدور:**
- البائع: المخزون المتاح فقط — لا أسعار تكلفة، لا حدود تنبيه
- admin/manager: العرض الكامل مع التكاليف والحدود

#### 10. العملاء — Clients

| البند | التفصيل |
|-------|---------|
| المسار | `/clients` (قائمة)، `/clients/[id]` (تفاصيل) |
| الأدوار | admin, manager |
| الغرض | إدارة قاعدة بيانات العملاء وتتبع ديونهم |

**الميزات الرئيسية:**
- إضافة عميل: اسم لاتيني + اسم عربي + هاتف + إيميل + عنوان + ملاحظات
- اكتشاف التكرار: تنبيه عند وجود عميل بنفس الاسم
- جدول العملاء: الاسم، الهاتف، إجمالي المبيعات، المدفوع، الدين المتبقي
- فلتر الديون: الكل / مع دين / بدون دين
- صفحة تفاصيل العميل:
  - ملخص العميل
  - سجل المبيعات الكامل
  - سجل الدفعات
  - نموذج دفع سريع (مبلغ + طريقة + اختيار بيعة محددة أو FIFO)
  - حساب TVA تلقائي
- بطاقات ملخص: عدد العملاء، إجمالي الديون
