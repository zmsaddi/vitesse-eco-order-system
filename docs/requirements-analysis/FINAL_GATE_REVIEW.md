## 8. اختبار إجهاد التغييرات المستقبلية

### إضافات آمنة (لا تكسر الأساس):
- إضافة فئات مصاريف جديدة
- إضافة فئات منتجات جديدة
- إضافة حقول اختيارية للعملاء/الموردين
- إضافة تقارير جديدة في Dashboard
- إضافة تصدير Excel
- إضافة إشعارات بريد إلكتروني
- إضافة صفحة تفاصيل المورد (مشابهة لتفاصيل العميل)
- إضافة دور خامس (مثل "محاسب")

### إضافات تحتاج حذرًا:
- إضافة عملة ثانية (يحتاج توسيع settings + كل الصيغ)
- إضافة نسب TVA مختلفة حسب المنتج (يحتاج عمود TVA في products)
- تغيير العلاقات النصية إلى FK بـ ID (migration كبير)

### المناطق الأكثر هشاشة:
- جدول `settings` (key-value بدون تحقق)
- العلاقات النصية (إذا أُضيفت المزيد)
- صيغة صافي الربح (إذا أُضيفت مصادر إيرادات أو تكاليف جديدة)
