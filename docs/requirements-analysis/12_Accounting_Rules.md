## طرق حساب الأرباح (3 طرق)

### 1. Pipeline (المتوقع)
- يشمل المبيعات المحجوزة + المؤكدة
- **الاستخدام**: تقدير الإيرادات المتوقعة

### 2. Accrual (الاستحقاق)
- يشمل المبيعات المؤكدة فقط (بعد التسليم)
- **الاستخدام**: المعيار المحاسبي — الإيراد يُعترف به عند التسليم

### 3. Cash Basis (النقدي)
- يشمل المبالغ المحصّلة فعلًا فقط
- **الاستخدام**: التدفق النقدي الحقيقي — أساس توزيع الأرباح

## حركة النقد (Cash Flow)

### التدفقات الداخلة (إيرادات)
| المصدر | الجدول | العمود | التفصيل |
|--------|--------|--------|---------|
| تحصيلات العملاء — كاش | payments | amount | WHERE type='collection' AND payment_method='كاش' |
| تحصيلات العملاء — بنك | payments | amount | WHERE type='collection' AND payment_method='بنك' |

### التدفقات الخارجة (مصروفات)
| المصدر | الجدول | العمود | التفصيل |
|--------|--------|--------|---------|
| دفعات الموردين — كاش | supplier_payments | amount | WHERE payment_method='كاش' |
| دفعات الموردين — بنك | supplier_payments | amount | WHERE payment_method='بنك' |
| مصاريف — كاش | expenses | amount | WHERE payment_type='كاش' |
| مصاريف — بنك | expenses | amount | WHERE payment_type='بنك' |

### صافي التدفق
```
صافي (كاش) = تحصيلات كاش - دفعات موردين كاش - مصاريف كاش
صافي (بنك) = تحصيلات بنك - دفعات موردين بنك - مصاريف بنك
```

## الدفعات الجزئية

### دفعات العملاء
- تحصيل جزئي مسموح للمبيعات الآجلة
- كل تحصيل ينشئ سجل `payments` مع TVA تناسبي
- المتبقي يُحدّث تلقائيًا في جدول `sales`
- FIFO: التحصيل من صفحة العميل يُطبّق على أقدم المبيعات أولًا

### دفعات الموردين
- دفع جزئي مسموح عند الإنشاء وبعده
- كل دفعة تُسجل في `supplier_payments`
- حالة الدفع تُحدّث تلقائيًا (pending → partial → paid)

---

## قواعد الاسترداد عند الإلغاء

1. **الدفعات**: لكل تحصيل سابق → سجل استرداد بمبلغ سالب
2. **TVA**: يُعكس (TVA سالب) لتصفير الميزان الضريبي
3. **المخزون**: يُعاد للمنتج
4. **العمولات**: تُحذف أو تُبقى حسب قرار المدير
5. **العمولات المصروفة**: تسوية سالبة (دين على الموظف)

---

## حساب TVA

- **النسبة**: من جدول `settings` (المفتاح: `vat_rate`، الافتراضي: 20%)
- **الصيغة**: `tva = amount × rate / (100 + rate)`
- **التطبيق**: يُستخرج من كل دفعة تحصيل ويُخزن في `payments.tva_amount`
- **الاسترداد**: TVA سالب عند إنشاء سجل استرداد

## تتبع الديون

### ديون العملاء
```
دين العميل = Σ(sales.remaining) WHERE client_name = X AND status ≠ 'ملغي'
```

### ديون الموردين
```
دين المورد = Σ(purchases.total - purchases.paid_amount) WHERE supplier = X AND payment_status ≠ 'paid'
```

## العملة والدقة المالية
- **العملة**: EUR (من `settings.invoice_currency`)
- **الدقة**: NUMERIC(19,2) — منزلتان عشريتان
- **التسامح الموحد**: 0.01€ في جميع المقارنات المالية (مبيعات + مشتريات + تسويات + تحقق حالة الدفع)
- **ملاحظة**: @vercel/postgres يُرجع NUMERIC كـ STRING — يجب parseFloat() قبل أي عملية حسابية
