# خريطة الحالات والانتقالات — State Transitions

> **رقم العنصر**: #7 | **المحور**: ب — سير العمل والقواعد | **الحالة**: مكتمل

---

## 1. حالات البيع (Sale Status)

```
                    ┌──────────┐
   إنشاء البيع ────▶│  محجوز   │ Reserved
                    └────┬─────┘
                         │
              ┌──────────┼──────────┐
              │ تأكيد التسليم       │ إلغاء
              ▼                     ▼
        ┌──────────┐          ┌──────────┐
        │  مؤكد    │ Confirmed│  ملغي    │ Cancelled
        └────┬─────┘          └──────────┘
             │                     ▲
             │ إلغاء (admin فقط)   │
             └─────────────────────┘
```

| من | إلى | الشرط | من يملك الصلاحية |
|----|-----|-------|-----------------|
| — | محجوز | إنشاء بيع جديد | admin, manager, seller |
| محجوز | مؤكد | تأكيد التسليم (تلقائي) | النظام عند تأكيد التوصيل |
| محجوز | ملغي | إلغاء البيع | admin, manager, seller (الخاص فقط) |
| مؤكد | ملغي | إلغاء البيع المؤكد (التعديل المباشر ممنوع — الإلغاء هو الإجراء الوحيد) | admin فقط |

## 2. حالات الدفع (Payment Status)

```
   ┌──────────┐
   │ pending  │ لم يُدفع
   └────┬─────┘
        │ تحصيل جزئي
        ▼
   ┌──────────┐
   │ partial  │ مدفوع جزئيًا
   └────┬─────┘
        │ اكتمال الدفع
        ▼
   ┌──────────┐
   │  paid    │ مدفوع بالكامل
   └──────────┘

   (من أي حالة → cancelled عند إلغاء البيع)
```

| من | إلى | الشرط |
|----|-----|-------|
| pending | partial | paid_amount > 0 و < total |
| partial | paid | paid_amount ≥ total (± 0.01) |
| pending | paid | دفع كامل دفعة واحدة |
| أي حالة | cancelled | إلغاء البيع |

**الصيغة**:
```
if paid_amount >= total - 0.01 → 'paid'
else if paid_amount > 0.01    → 'partial'
else                           → 'pending'
```

## 3. حالات التوصيل (Delivery Status)

```
   ┌─────────────┐
   │ قيد الانتظار │ Pending
   └──────┬──────┘
          │ تعيين سائق / بدء التوصيل
          ▼
   ┌─────────────┐
   │جاري التوصيل │ In Transit
   └──────┬──────┘
          │
    ┌─────┴──────┐
    ▼            ▼
┌────────┐  ┌────────┐
│تم التوصيل│  │ ملغي   │
│Delivered│  │Cancelled│
└────────┘  └────────┘
```

| من | إلى | الشرط | من يملك الصلاحية |
|----|-----|-------|-----------------|
| — | قيد الانتظار | إنشاء توصيل (تلقائي مع البيع) | النظام |
| قيد الانتظار | جاري التوصيل | السائق يبدأ التوصيل | admin, manager, driver |
| جاري التوصيل | تم التوصيل | تأكيد التسليم (+ VIN للدراجات) | admin, manager, driver |
| قيد الانتظار | ملغي | إلغاء (عبر إلغاء البيع) | admin |
| جاري التوصيل | ملغي | إلغاء مع تأكيد صريح + سبب إلزامي | admin |

## 4. حالات دفع المشتريات (Purchase Payment Status)

```
   ┌──────────┐
   │ pending  │ لم يُدفع للمورد
   └────┬─────┘
        │ دفعة جزئية
        ▼
   ┌──────────┐
   │ partial  │ مدفوع جزئيًا
   └────┬─────┘
        │ اكتمال الدفع
        ▼
   ┌──────────┐
   │  paid    │ مدفوع بالكامل
   └──────────┘
```

| من | إلى | الشرط |
|----|-----|-------|
| pending | partial | دفعة أولى > 0 و < الإجمالي |
| pending | paid | دفع كامل عند الإنشاء |
| partial | paid | اكتمال الدفعات الجزئية |

## 5. حالات الفاتورة (Invoice Status)

```
   ┌──────────┐
   │  مؤكد    │ Confirmed (افتراضي عند الإنشاء)
   └────┬─────┘
        │ إلغاء البيع
        ▼
   ┌──────────┐
   │  ملغي    │ Voided
   └──────────┘
```

| من | إلى | الشرط |
|----|-----|-------|
| — | مؤكد | إنشاء تلقائي عند تأكيد التسليم |
| مؤكد | ملغي | إلغاء ناعم (invoiceMode='soft') |
| مؤكد | — | حذف كامل (invoiceMode='delete') |

## 6. حالات العمولة (Bonus Status)

```
   ┌──────────────┐
   │ غير مصروف    │ settled = false
   │  (مستحق)     │
   └──────┬───────┘
          │ تسوية
          ▼
   ┌──────────────┐
   │  مصروف      │ settled = true
   │ (تم الصرف)   │ settlement_id مرتبط
   └──────────────┘
```

| من | إلى | الشرط |
|----|-----|-------|
| — | غير مصروف | حساب تلقائي عند تأكيد التسليم |
| غير مصروف | مصروف | تسوية من المدير (FIFO) |
| مصروف | — | عند الإلغاء: تسوية سالبة (دين استرداد) |

        │ معالجة ناجحة

## 8. حالات حساب المستخدم (User Active Status)

```
   ┌──────────┐
   │  مفعّل   │ active = true (افتراضي)
   └────┬─────┘
        │ تعطيل بواسطة المدير
        ▼
   ┌──────────┐
   │  معطّل   │ active = false
   └────┬─────┘
        │ إعادة تفعيل
        ▼
   ┌──────────┐
   │  مفعّل   │ active = true
   └──────────┘
```

**ملاحظة**: المستخدم المعطّل لا يستطيع تسجيل الدخول، لكن عمولاته غير المصروفة تبقى قابلة للتسوية.
