## أنواع الإشعارات الحالية

### 1. إشعارات Toast (داخل النظام)
- **المكون**: Toast.js مع useToast() hook
- **المدة**: 3 ثوانٍ ثم إخفاء تلقائي
- **الأنواع**: success (أخضر) / error (أحمر)

#### رسائل النجاح
| الرسالة | السياق |
|---------|--------|
| تم إضافة العميل بنجاح | إنشاء عميل |
| تم حفظ التعديلات | تعديل أي سجل |
| تم الحذف بنجاح | حذف سجل |
| تم إلغاء الطلب بنجاح | إلغاء بيع |
| تم تسجيل الدفعة بنجاح | تحصيل دفعة |
| تم إضافة المستخدم | إنشاء مستخدم |
| تم تحديث المستخدم | تعديل مستخدم |
| تم حفظ معدلات العمولة المخصصة | تعيين تجاوز عمولة |

#### رسائل الخطأ
| الرسالة | السياق |
|---------|--------|
| خطأ في جلب البيانات | فشل تحميل |
| خطأ في إضافة البيانات | فشل إنشاء |
| خطأ في تعديل البيانات | فشل تحديث |
| خطأ في الحذف | فشل حذف |
| خطأ في الاتصال | فشل شبكة |

### 2. تنبيه المخزون المنخفض
- **الآلية**: عرض مرئي في صفحة المخزون (badge "منخفض" أو "نفذ")
- **الحد**: `low_stock_threshold` لكل منتج (افتراضي: 3)
- **الأدوار**: admin, manager فقط
- **لا إشعار push**: يظهر فقط عند زيارة صفحة المخزون

### 3. نوافذ التأكيد (ConfirmModal)
- تظهر قبل العمليات التدميرية (حذف، إلغاء، تغيير دور، تغيير سعر)
- تتطلب تأكيد صريح من المستخدم
