## 3. قواعد الحذف حسب الكيان

### المنتج (Product)
- ❌ ممنوع إذا stock > 0
- ❌ ممنوع إذا له مبيعات أو مشتريات مرتبطة
- ✅ مسموح فقط إذا stock = 0 ولا سجلات مرتبطة

### العميل (Client)
- ❌ ممنوع إذا له مبيعات غير ملغية
- ✅ مسموح فقط إذا لا مبيعات

### المورد (Supplier)
- ❌ ممنوع إذا له مشتريات أو سجلات دفعات مرتبطة (مطابق لنمط حماية العملاء)
- رسالة: "لا يمكن حذف المورد — يوجد مشتريات مرتبطة به"

### المشترى (Purchase)
- ❌ ممنوع إذا المخزون الحالي < كمية المشترى
- ✅ يعكس: المخزون + المتوسط المرجح + سجل أسعار

### المستخدم (User)
- ❌ الحذف النهائي ممنوع — يُستخدم التعطيل فقط (active=false)
- السبب: حذف المستخدم يترك عمولات وتسويات وسجلات تدقيق يتيمة
- المستخدم المعطّل لا يدخل النظام لكن جميع سجلاته التاريخية تبقى سليمة

## 4. القاعدة الذهبية

> **ممنوع منعًا باتًا** تنفيذ أي عملية reset أو truncate أو cleanup على قاعدة بيانات الإنتاج.
> قاعدة البيانات تحتوي بيانات مسجلة حقيقية.

- **التنفيذ**: `ALLOW_DB_RESET=false` + `NODE_ENV` check
- **التأكيد**: عبارة "احذف كل البيانات نهائيا" مطلوبة

## 5. الحذف الناعم vs الحذف الصلب

| الكيان | نوع الحذف | التفصيل |
|--------|----------|---------|
| البيع الملغي | ناعم | status = 'ملغي' — السجل يبقى |
| التوصيل الملغي | ناعم | status = 'ملغي' — السجل يبقى |
| الفاتورة الملغية | ناعم (soft) | status = 'ملغي' — السجل يبقى |
| الفاتورة (delete mode) | صلب | الصف يُحذف نهائيًا |
| المصاريف | صلب | الصف يُحذف نهائيًا |
| المشتريات | صلب | الصف يُحذف مع عكس المخزون |
