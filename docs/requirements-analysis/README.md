# مواصفات النظام — VITESSE ECO SAS v2

## نظام إدارة الطلبات والعمليات — Order & Operations Management System

> **الحالة**: قيد التطوير — توثيق قبل التنفيذ
> **آخر تحديث**: 2026-04-18
> **اللغات**: العربية (واجهة + مصطلحات تجارية) + الإنجليزية (مصطلحات تقنية)
> **الفرنسية**: فقط في الفواتير المُولّدة (متطلب قانوني)
> **الدقة المالية**: 0.01€ — كل الأرقام TTC
> **المنطقة الزمنية**: Europe/Paris
> **مشروع جديد**: لا ترحيل بيانات من v1

---

## الرؤية

نظام متكامل لإدارة عمليات البيع والشراء والتوصيل والمخزون والمالية لشركة بيع مركبات صديقة للبيئة في فرنسا. يدعم 6 أدوار وظيفية، طلبات متعددة الأصناف، صناديق مالية هرمية، كتالوج منتجات بصور، ونظام تنبيهات فوري. مُصمم لتجربة موحدة على الكمبيوتر والتابلت والموبايل، مع تجهيز لتطبيق Android (React Native) لاحقاً.

## القرارات المعتمدة

### قرارات حرجة

| # | القرار |
|---|--------|
| C1 | **شاشة إلغاء موحدة:** 3 خيارات إلزامية (مخزون / عمولة سائق / عمولة بائع). لا قيم افتراضية. عمولة مصروفة → دين. |
| C2 | **الإلغاء مسموح دائماً.** العمولات المصروفة → دين على المستلم. |
| C3 | Rate limiting مؤجل (10-20 مستخدم). |
| C4 | **نسخ احتياطي يدوي:** زر تنزيل/استعادة لـ PM/GM. |
| C5 | **حذف مشتريات:** سؤال "هل استُرد المبلغ؟" نعم→ صندوق. لا→ رصيد دائن للمورد. |

### قرارات عالية

|---|--------|
| H1 | **كل الأرقام TTC.** TVA فقط عند توليد الفاتورة. |

| H3 | تعديل كمية/سعر مسموح + تحذير فقط. |
| H4 | تحديث شامل ذري عند تغيير اسم كيان. |

## قواعد التوثيق

- جميع الوثائق بصيغة Markdown
- **العربية** للمصطلحات التجارية والواجهة
- **الإنجليزية** للمصطلحات التقنية والكود
- **الفرنسية** في: قالب الفاتورة (SIRET, TVA, TTC/HT, mentions légales) + كتالوج PDF (اختياري)
- **كتالوج PDF** يمكن توليده بـ 3 لغات: العربية / الإنجليزية / الفرنسية (يختار المستخدم)
- التوثيق يُحدّث **قبل** كتابة أي كود
- كل ملف يحتوي رقم العنصر + المحور + الحالة
