## 1. النسخ الاحتياطي (Neon)

| البند | التفصيل |
|-------|---------|
| المزود | Neon Serverless PostgreSQL |
| النسخ التلقائي | Neon يوفر point-in-time recovery (PITR) |
| الاحتفاظ | حسب خطة Neon (عادةً 7-30 يوم) |
| الاستعادة | عبر لوحة تحكم Neon — استعادة لأي نقطة زمنية |
| التشفير | اتصال SSL مطلوب (sslmode=require) |

## 2. حماية من الحذف العرضي

### بوابة ALLOW_DB_RESET
- **المتغير**: `ALLOW_DB_RESET` في `.env`
- **الافتراضي**: `false`
- **في الإنتاج**: يجب أن يكون `false` دائمًا
- **التأثير**: POST /api/init مع action='reset' يُرفض إذا:
  - `NODE_ENV === 'production'`
  - أو `ALLOW_DB_RESET !== 'true'`
- **عبارة التأكيد**: "احذف كل البيانات نهائيا" مطلوبة كتأكيد إضافي

### الحذف الناعم
- المبيعات والتوصيلات والفواتير الملغية تبقى بحالة 'ملغي' — لا تُحذف فعليًا
- سجلات التدقيق (cancellations, price_history, supplier_payments) لا تُحذف أبدًا

### حماية FK
- ON DELETE CASCADE على العمولات والفواتير (تُحذف مع البيع/التوصيل)
- ON DELETE SET NULL على التوصيل (يبقى بدون بيع)

---

## 3. القاعدة الذهبية

> **ممنوع منعًا باتًا** تنفيذ أي من العمليات التالية على قاعدة بيانات الإنتاج:
> - `TRUNCATE TABLE`
> - `DROP TABLE`
> - `DELETE FROM table` (بدون WHERE)
> - `UPDATE table SET` (بدون WHERE)
> - أي script يمسح أو يعيد تعيين البيانات
>
> **السبب**: قاعدة البيانات تحتوي بيانات مسجلة حقيقية لمستخدمين حقيقيين.

## 4. ما لا يوجد حاليًا

| البند | الحالة |
|-------|--------|
| نسخ احتياطي خارجي أسبوعي | **مطلوب**: pg_dump أسبوعي إلى تخزين خارجي (S3 أو ما يعادله) |
| نسخ خارجي (S3, GCS) | غير موجود |
| تصدير دوري للبيانات | غير موجود |
| خطة استعادة الكوارث (DR) | يعتمد على Neon PITR |
| تشفير البيانات أثناء الراحة | يعتمد على Neon |
