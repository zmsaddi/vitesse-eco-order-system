# قواعد رؤية البيانات — Data Visibility Rules

> **رقم العنصر**: #15 | **المحور**: ج | **الحالة**: مكتمل

---

## 1. فلترة البيانات حسب المستخدم

### المبيعات (GET /api/sales)
- **admin/manager**: جميع المبيعات
- **seller**: `WHERE created_by = username` (مبيعاته فقط)
- **driver**: المبيعات المرتبطة بتسليماته

### التوصيل (GET /api/deliveries)
- **admin/manager**: جميع التسليمات
- **seller**: `WHERE created_by = username`
- **driver**: `WHERE assigned_driver = username` (المعينة له فقط)

### الفواتير (GET /api/invoices)
- **admin/manager**: جميع الفواتير
- **seller**: فواتير مبيعاته فقط
- **driver**: فواتير تسليماته فقط

### العمولات (GET /api/bonuses)
- **admin/manager**: جميع العمولات (للإشراف على الرواتب)
- **seller/driver**: `WHERE username = token.username` (الخاصة فقط)

### لوحة التحكم (GET /api/summary)
- **admin/manager**: P&L كامل، التدفق النقدي، جميع المؤشرات
- **seller**: إحصائيات شخصية فقط (مبيعاتي، عمولتي، الدين المتبقي)

---

## 2. إخفاء الأعمدة حسب الدور

### سعر التكلفة (buy_price)
- **admin/manager**: مرئي في المنتجات + المبيعات + لوحة التحكم
- **seller**: **مخفي تمامًا** — يُزال من استجابة GET /api/products
  ```javascript
  products.map(({ buy_price, ...rest }) => rest)
  ```
- **رسائل خطأ مختلفة**: البائع لا يعرف سعر التكلفة حتى في رسائل الخطأ

### أعمدة الربح (cost_price, cost_total, profit)
- **admin/manager**: مرئية في جدول المبيعات
- **seller**: مخفية — لا تظهر كأعمدة في الجدول

### تفاصيل التكلفة في لوحة التحكم
- **admin/manager**: تكلفة البضاعة، الربح الإجمالي، قيمة المخزون بالتكلفة
- **seller**: إجمالي المبيعات والعمولات فقط

### حد التنبيه (low_stock_threshold)
- **admin/manager**: مرئي وقابل للتعديل
- **seller**: مخفي — يرى فقط "متوفر / منخفض / نفذ"

---

## 3. البيانات المحدودة عند الإنشاء

### المنتجات (POST /api/products)
- **seller**: يمكنه إنشاء منتج shell فقط (الأسعار والمخزون = 0)
- **admin/manager**: إنشاء كامل مع جميع البيانات
