## 1. سير عمل المشتريات (Purchase Workflow)

```
المدير/المشرف يدخل بيانات المشترى
        │
        ▼
┌─────────────────────────┐
│ التحقق من البيانات       │ Zod schema validation
│ - المورد مطلوب           │
│ - المنتج + الفئة مطلوبة  │
│ - الكمية > 0             │
│ - سعر الوحدة > 0         │
│ - سعر البيع ≥ سعر الشراء │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│ قفل صف المنتج FOR UPDATE │
│ (منع التعديل المتزامن)   │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│ هل المنتج موجود؟         │
│ نعم → تحديث المتوسط المرجح│
│ لا  → إنشاء منتج جديد    │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│ تحديث المخزون: +كمية     │
│ تحديث buy_price (وزني)   │
│ تحديث sell_price (إن وُجد)│
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│ تسجيل سجل الأسعار       │
│ price_history ← قديم/جديد│
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│ هل الدفعة الأولى > 0؟    │
│ نعم → إنشاء supplier_payment│
│ لا  → حالة الدفع = pending│
└────────┬────────────────┘
         ▼
    ✅ مشترى مسجل + ref_code
```

## 2. سير عمل المبيعات (Sale Workflow)

```
البائع/المدير يدخل بيانات البيع
        │
        ▼
┌──────────────────────────────┐
│ التحقق من البيانات            │
│ - العميل + المنتج مطلوبان     │
│ - الكمية > 0                  │
│ - سعر البيع ≥ buy_price (الكل)│
│ - سعر البيع ≥ sell_price (بائع)│
│ - كاش/بنك: الدفعة = الإجمالي  │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ قفل صف المنتج FOR UPDATE     │
│ التحقق: الكمية ≤ المخزون     │
│ خصم المخزون فورًا (حجز)      │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ هل العميل موجود؟              │
│ نعم → تحديث بيانات العميل     │
│ لا  → إنشاء عميل جديد تلقائي │
│       (مع التحقق من التكرار)  │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ إدراج سجل البيع              │
│ status = 'محجوز'              │
│ payment_status = 'pending'    │
│ ref_code = SL-YYYYMMDD-XXXXX │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ إنشاء سجل توصيل مرتبط تلقائيًا│
│ status = 'قيد الانتظار'       │
│ sale_id = البيع الجديد        │
│ ref_code = DL-YYYYMMDD-XXXXX │
└───────────┬──────────────────┘
            ▼
    ✅ بيع مسجل + توصيل جاهز
```

## 3. سير عمل التوصيل (Delivery Workflow)

```
المدير يعيّن سائق للتوصيل
        │
        ▼
┌──────────────────────────┐
│ السائق يحدّث الحالة       │
│ قيد الانتظار → جاري التوصيل│
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ السائق يؤكد التسليم       │
│ جاري التوصيل → تم التوصيل │
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ هل المنتج دراجة/سكوتر؟   │
│ نعم → VIN مطلوب (نافذة)   │
│ لا  → متابعة              │
└───────────┬──────────────┘
            ▼
  ┌─────────┴──────────────────────────────────┐
  │ 4 عمليات تلقائية متسلسلة (معاملة ذرية):    │
  │                                             │
  │ 1. تأكيد البيع → status = 'مؤكد'           │
  │ 2. تحصيل الدفعة المقدمة (إن > 0)           │
  │ 3. إنشاء فاتورة → INV-YYYYMM-NNN          │
  │ 4. حساب العمولات (بائع + سائق)             │
  └─────────┬──────────────────────────────────┘
            ▼
    ✅ تسليم مكتمل + فاتورة + عمولات
```

## 4. سير عمل التحصيل (Payment Collection Workflow)

```
المدير يفتح صفحة تفاصيل العميل أو البيع
        │
        ▼
┌────────────────────────────┐
│ اختيار طريقة التحصيل:       │
│ أ) بيع محدد: /sales/[id]/collect│
│ ب) FIFO: /clients/[id]/collect │
└───────────┬────────────────┘
            ▼
┌────────────────────────────┐
│ إدخال المبلغ + طريقة الدفع │
│ التحقق: المبلغ ≤ المتبقي   │
│ التحقق: البيع = مؤكد       │
└───────────┬────────────────┘
            ▼
┌────────────────────────────┐
│ حساب TVA تلقائي:           │
│ tva = amount × rate / (100+rate)│
└───────────┬────────────────┘
            ▼
┌────────────────────────────┐
│ إدراج سجل دفعة (payment)   │
│ type = 'collection'         │
│ تحديث البيع:               │
│   paid_amount += المبلغ     │
│   remaining -= المبلغ       │
│   payment_status تلقائي     │
└───────────┬────────────────┘
            ▼
    ✅ دفعة مسجلة
```

## 5. سير عمل الإلغاء (Cancellation Workflow — 12 خطوة)

```
المدير يطلب إلغاء البيع
        │
        ▼
┌──────────────────────────────┐
│ 1. قفل صف البيع FOR UPDATE   │
│ 2. التحقق: هل ملغي مسبقًا؟   │
│    نعم → خطأ: "مُلغى مسبقًا" │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ 3. التحقق من العمولات المصروفة│
│    مصروفة → خطأ 409          │
│ 4. هل bonusActions محددة؟     │
│    لا → خطأ 428 + معاينة     │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ 5. استرداد الدفعات:          │
│    لكل تحصيل → سجل استرداد   │
│    (مبلغ سالب + TVA سالب)    │
│ 6. استعادة المخزون: +كمية    │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ 7. معالجة العمولات:          │
│    'remove' → حذف + تسوية سالبة│
│    'keep' → بدون تغيير       │
│ 8. التوصيل → حالة 'ملغي'     │
│ 9. الفاتورة → إلغاء ناعم/حذف │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ 10. البيع → حالة 'ملغي'      │
│     paid_amount = 0           │
│     remaining = 0             │
│     payment_status = cancelled│
│ 11. تسجيل في cancellations   │
│ 12. إرجاع النتيجة             │
└───────────┬──────────────────┘
            ▼
    ✅ إلغاء مكتمل مع سجل تدقيق
```

## 6. سير عمل التسوية (Settlement Workflow)

```
المدير يفتح صفحة التسويات
        │
        ▼
┌──────────────────────────┐
│ اختيار النوع:             │
│ دفع عمولة بائع / سائق    │
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ النظام يجلب المستخدمين    │
│ المؤهلين مع الرصيد المتاح │
│ (عمولات غير مصروفة)       │
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ إدخال: المستخدم + المبلغ  │
│ التحقق: المبلغ ≤ الرصيد   │
│ إعادة التحقق قبل الإرسال  │
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ قفل العمولات FOR UPDATE   │
│ وسم FIFO: settled = true  │
│ من الأقدم للأحدث          │
│ ربط settlement_id         │
└───────────┬──────────────┘
            ▼
    ✅ تسوية مسجلة + عمولات وُسمت
```

## 7. سير عمل توزيع الأرباح (Profit Distribution Workflow)

```
المدير يفتح صفحة توزيع الأرباح
        │
        ▼
┌────────────────────────────────┐
│ تحديد الفترة (اختياري)         │
│ النظام يحسب:                   │
│ - صافي الربح النقدي للفترة     │
│ - الموزع سابقًا               │
│ - المتبقي للتوزيع              │
└───────────┬────────────────────┘
            ▼
┌────────────────────────────────┐
│ إدخال المبلغ الأساسي           │
│ أو "تحميل من الإعدادات"        │
│ إضافة مستلمين + نسب مئوية      │
│ التحقق: مجموع النسب = 100%     │
└───────────┬────────────────────┘
            ▼
┌────────────────────────────────┐
│ قفل استشاري (advisory lock)    │
│ على hash الفترة                │
│ التحقق: المبلغ ≤ المتاح        │
└───────────┬────────────────────┘
            ▼
┌────────────────────────────────┐
│ إنشاء group_id واحد           │
│ إدراج صف لكل مستلم:           │
│   username + percentage + amount│
└───────────┬────────────────────┘
            ▼
    ✅ توزيع مسجل
```

## 8. سير عمل دفعة المورد (Supplier Payment Workflow)

```
المدير يفتح مشترى بحالة partial/pending
        │
        ▼
┌──────────────────────────┐
│ الضغط على زر "دفع"       │
│ عرض: الإجمالي، المدفوع، المتبقي│
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ إدخال: المبلغ + طريقة الدفع│
│ التحقق: المبلغ ≤ المتبقي  │
└───────────┬──────────────┘
            ▼
┌──────────────────────────┐
│ قفل صف المشترى FOR UPDATE │
│ تحديث paid_amount         │
│ تحديث payment_status      │
│ إدراج supplier_payment    │
└───────────┬──────────────┘
            ▼
    ✅ دفعة مورد مسجلة
```

## 9. سير عمل تعديل بيع (Sale Edit Workflow)

### 9أ. تعديل بيع قبل التسليم (محجوز) — مسموح

```
admin/manager/seller يفتح بيع بحالة "محجوز"
        │
        ▼
┌──────────────────────────────┐
│ تعديل: السعر / الكمية /      │
│ المنتج / العميل / الملاحظات  │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ هل تغير المنتج أو الكمية؟    │
│ نعم → إعادة حجز المخزون:     │
│       +كمية قديمة → −كمية جديدة│
│ لا  → بدون تغيير مخزون       │
└───────────┬──────────────────┘
            ▼
┌──────────────────────────────┐
│ created_by لا يتغير           │
│ (الملكية تبقى للمنشئ الأصلي)  │
│ updated_by = المُعدّل         │
│ لا عمولات تُحسب (قبل التسليم) │
└───────────┬──────────────────┘
            ▼
    ✅ بيع مُعدّل — بدون تأثير مالي
```

### 9ب. تعديل بيع بعد التسليم (مؤكد) — ممنوع

```
admin يفتح بيع بحالة "مؤكد"
        │
        ▼
┌──────────────────────────────┐
│ ❌ التعديل المباشر ممنوع      │
│ البيع مرتبط بفاتورة + عمولات │
│                              │
│ الإجراء الوحيد المتاح:        │
│ → إلغاء البيع (12 خطوة)      │
│ → إنشاء بيع جديد بالقيم الصحيحة│
└──────────────────────────────┘
```
